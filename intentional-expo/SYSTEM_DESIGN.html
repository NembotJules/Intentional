Local-first attention ledger.

Intentional connects pillars to daily actions to logged focus time. The system should keep that ledger honest, visually calm, and testable before adding ambient or monetization features.

Ship iPhone-first v1. Android and web remain development or demo surfaces until explicitly promoted.

Design References Impact

They Do Affect Architecture

The references require stateful surfaces: empty Today, partial sessions, shield denied, note receipt, uneven Insights, archive confirmation, and reminder permission states. These become test and route requirements.

They Do Not Replace Product Logic

The data graph remains pillars -> actions -> sessions. The references change how that graph is presented, what states must be represented, and which shared components should exist.

Focus Is Special

Active Focus is a mode change, not another tab screen. It uses dark shell, reduced chrome, honest shield copy, and no bottom nav while active.

Screen State Contract

Screen Required States From References Architectural Implication
Today Mixed actions, empty day, partial logged session. Today query layer must return grouped pillar/action progress and support manual focus entry.
Focus Pre-session, active shielded, paused, shield denied. Focus state machine must model shield status separately from timer status.
Session Complete Natural completion, ended early, short note. Session persistence must happen before note editing; partial time always counts.
Insights Populated week, empty ledger, uneven pillar week. Aggregates need readable totals plus plain-language summary inputs.
Goals List with weekly evidence, lightweight detail, archive confirmation. Archive remains soft delete. Historical sessions stay queryable.
Onboarding Pillar selection, first action, optional why. Onboarding should create a usable Today quickly. Draft persistence is useful but should not own product state.
Reminders Action edit setup, permission prompt, disabled state. Reminder service needs permission-denied UX and action-specific notification identifiers.
